기술분야 상세내용
컬렉션 List<> 인터페이스
List<>와 ArrayList<> 클래스를 활용하여 데이터를 DB에 DML 할 때 사용했다.
JSTL JSP 페이지 내에서 스크립틀릿을 사용하지 않고 JSTL을 사용하여 자바 코드를 바로 사용하지 않고 로직을 내장하였다.
MVC 패턴 JSP 파일 내에서 자바 코드를 사용하지 않고 모델, 뷰, 컨트롤러로 각각 페이지를 나누어 코드를 작성함으로써 가독성을 높이고, 오류 발생 시 오류 수정에 시간을 아낄 수 있도록 함
JavaScript JSP에서 조건 체크 시 JavajScript를 사용하였다.
Ajax JSP에서 조건 체크 시 Ajax를 사용하여 컨트롤러로 데이터를 주고받기 위해 사용하였다.
jQuery JavaScript의 라이브러리인 jQuery를 활용하여 코드를 간략히 하였다.
Kakao API 지도 API를 사용하여 숙소의 위치를 확인할 수 있다.
느낀점
git도 점점 익숙해지고 MVC 패턴과 스프링에 대한 이해도 많이 발전할 수 있었던 프로젝트였습니다. 이전 프로젝트에서는 다른 팀원들과 겹치는 부분이 별로 없었기 때문에 코드가 서로 섞이지 않았지만, 이번에는 겹치는 부분이 많아 팀원들과 소통이 중요했는데 원활히 소통이 되어서 보다 쉽게 프로젝트를 끝낼 수 있었습니다. 기본적인 CRUD는 어려움 없이 진행했고 새로운 api들이나 이전보다 더 깔끔하게 코딩하려고 노력했습니다. 테이블들이 많고 아직 효율적으로 테이블을 짜는 게 어려워 골치 아프긴 했지만 문제없이 기능들을 구현해서 재미있었고, 여러 오류들을 만나면서 정리를 해놓은 덕분에 오류의 원인과 해결하는 시간을 많이 단축시킬 수 있었습니다.
