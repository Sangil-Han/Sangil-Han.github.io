기술분야 상세내용
컬렉션 List<>, Map<> 인터페이스
List<> 를 활용하여 데이터를 DB에 DML 할 때 사용했고, HashMap<> 을 사용해서 검색 필터링 후 페이징 처리를 하였다.
JSTL JSP 페이지 내에서 스크립틀릿을 사용하지 않고 JSTL을 사용하여 자바 코드를 바로 사용하지 않고 로직을 내장하였다.
MVC 패턴 JSP 파일 내에서 자바 코드를 사용하지 않고 모델, 뷰, 컨트롤러로 각각 페이지를 나누어 코드를 작성함으로써 가독성을 높이고, 오류 발생 시 오류 수정에 시간을 아낄 수 있도록 함
JavaScript JSP에서 조건 체크 시 JavajScript를 활용하였다.
Spring MultipartFile multipartResolver 빈 등록 후 게시글 등록 시 파일 첨부를 위해 사용하였다.
Spring ModelAndView ModelAndView의 addObject()를 값을 넣을 때 사용하였고, setViewName()으로 View의 경로를 설정하였다.
느낀점
Spring 프레임워크를 이용한 첫 프로젝트였습니다. 지금까지 공부해왔던걸 종합적으로 알맞게 사용해야 했기에 공부가 많이 됐었고, 어려웠던 점은 게시글 목록에서 검색 기능과 페이징 처리를 같이 구현하는 것이었습니다. RowBounds 클래스와 HashMap 클래스를 이용해서 구현했는데 비교적 복잡하다고 생각이 들었고, 프론트엔드, 백엔드 언어를 사용하면서 구현하다 보니 이렇게 작은 게시판 사이트 하나를 만드는데 여러 오류들을 만나 해결하며, 많은 시간, 노력이 들어간다는 걸 느끼면서 오류 해결 능력과 앞으로 문제 해결에 대한 자신감을 키우는 데 도움이 많이 된 프로젝트입니다. 첫 프로젝트를 진행하면서 여러 많은 것들을 정리하고 흥미를 느낀 계기가 되었고, 이 프로젝트가 다음 프로젝트에서 더욱 효율적이고 발전된 프로그래밍을 위한 좋은 원동력이 될 거라 생각합니다.
